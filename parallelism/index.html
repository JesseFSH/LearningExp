Ansible使用vmware-guest module进行克隆虚机可以实现5台并发安装虚机（ansible默认并发为5），但是自己用python调用vmware pyvmomi接口安装虚机没有实现并发功能，实现逻辑是for循环。考虑修改为并发安装。python并发编程主要涉及两个库Threading和Multiprocessing。 https://chriskiehl.com/article/parallelism-in-one-line 发现一篇文章，开始啃。文章里有几块代码，保存在本地了。 C:\Users\Pactera\PycharmProjects\pythonProject\parallelism.py

  1. 对于类class中的def init(self,arg1,arg2,...)方法，init__的作用是当类被实例化的时候，会自动调用__init__方法对实例进行初始化，实例化类的时候应该这么写obj = class(arg1,arg2)，__init__内除了self以外的参数需要被声明出来。obj是在类外部，对类的实例化，而在类的内部，则用self来指代类的实例。具体解释可以参考下面链接。 https://micropyramid.com/blog/understand-self-and-__init-method-in-python-class/#:~:text=%22__init__%22%20is%20a%20reseved%20method%20in%20python%20classes.,class%20to%20initialize%20the%20attributes%20of%20a%20class.
  2. isinstance()函数用来判断一个对象是否是一个已知的类型，类似 type()。
  3. threading.Thread.join()方法，会等待主线程结束后，再结束子线程。
  4. Consumer类继承了Threading.thread父类，并对父类的run方法进行了重写
  5. 第二个案例引入了def build_worker_pool(queue, size)这个函数，通过这个函数增加了多个线程。
  6. 最后引入了multiprocessing.Pool.map/multiprocessing.dummy.Pool.map这两个方法来完成python并发，map方法定义： def map(self, func, iterable, chunksize=None) 第一个参数为func，第二个参数为可遍历的类型，例如列表和元组
