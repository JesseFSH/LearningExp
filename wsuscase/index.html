  • WSUS报错0x80244010

2021.12.8 wsus case

这个问题困扰了很久，经常遇到服务器check for updates一直读条，最后报错。甚至是新装服务器也有这样的情况。

case现象

WindowsUpdate.log有如下报错特征：

  • WARNING: WU client failed Searching for update with error 0x80244010
  • WARNING: Exceed max server round trips: 0x80244010 + SyncUpdates round trips:201

官方KB报错描述参考：https://docs.microsoft.com/en-us/windows/deployment/update/windows-update-error-reference

解决方案

  1. 备份所有WSUS（上游+下游）数据库SUSDB，可以通过SQL Server Management Studio进入数据库备份。
  2. 将所有被取代的更新拒绝。
  3. WSUS控制台->选项->服务器清理向导Server Cleanup Wizard
  4. 客户端停止Windows Update服务，重命名C:\Windows\SoftwareDistribution文件夹，启动Windows Update服务并运行check for updates
